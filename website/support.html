How does text extraction and OCR work in Paperwise, and what options can I choose?

Paperwise first tries to extract text directly from the document when possible. That works well for machine-readable PDFs and avoids unnecessary OCR. When the file is a scan, image-heavy PDF, or low-quality document, Paperwise falls back to OCR based on the mode you choose in Settings.

There are three main OCR paths:

Local Tesseract keeps OCR on your machine using tesseract and pdftoppm. This is a good default for privacy-sensitive setups and clean printed scans, but it can struggle more with messy layouts, handwriting, or degraded images.

LLM OCR using your main model connection sends rendered page images to the same model family you use for the rest of the pipeline. This is convenient when you want one shared provider setup.

Separate OCR model routing lets you pick a dedicated OCR model independently from metadata extraction and Q&A. This is usually the best setup when you want a fast multimodal model for OCR and a different model for structured extraction or reasoning.

For OpenAI-based vision OCR, you can also tune image detail to auto, low, or high. Higher detail can improve difficult scans, but it may increase latency and cost. Paperwise also supports an auto-switch mode so OCR is only used when local extraction looks weak.

Which AI models should I use for OCR, extraction, and Q&A?

Paperwise works well with both GPT and Gemini setups. Pick the provider you prefer, then use task-specific models so OCR, extraction, and grounded Q&A are tuned separately.

Task GPT Recommendation Gemini Recommendation Notes
OCR gpt-5-mini gemini-2.5-flash Best fast multimodal options for scanned PDFs, forms, and image-heavy documents. For harder cases like handwriting, consider stepping up to a pro-tier model.
Metadata extraction gpt-5-mini gemini-2.5-flash Balanced choices for structured fields like title, date, correspondent, and document type.
Grounded Q&A gpt-5.1 gemini-2.5-pro Stronger picks for harder questions that require synthesis across many chunks.
Budget / bulk work gpt-5-nano gemini-2.5-flash-lite Good for lightweight classification, triage, and high-throughput pipelines.

If your documents are mostly clean text PDFs, start with the faster / cheaper row and only move up for tougher Q&A or more demanding extraction quality. See the latest provider docs for current model availability: OpenAI models and Gemini models.

Why does OCR sometimes time out on dense PDFs?

Large page images can take longer to process. Paperwise logs per-page OCR progress and retries timed-out pages. Longer OCR request timeouts are enabled for vision calls.

How are duplicate files handled?

Documents are deduplicated by SHA256 checksum to prevent repeated ingestion of the same file content.
